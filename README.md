## 项目结构

项目结构如下：

```
-- api          接口模块
-- components   组件模块
-- layouts      布局模块
-- middleware   中间件模块
-- pages        页面模块
-- plugins      插件模块
-- static       静态资源模块
-- store        状态管理模块
-- utils        工具模块
-- types        类型定义模块
```

## git提交规范

```
例如：【docs】什么进行了什么

feat - 新功能 feature

fix - 修复 bug

docs - 文档注释

style - 代码格式(不影响代码运行的变动)

refactor - 重构、优化(既不增加新功能，也不是修复bug)

perf - 性能优化

test - 增加测试

revert - 回退
```
